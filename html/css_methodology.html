CSS-Methodology

Our main focus lies in using readable, maintainable and future proof CSS. Therefore we use SCSS "SASSY CSS" and BEM "Block Element Modifier" for our CSS.

Class Naming Convention

We use the Block Element Naming Conventions. Blocks are standalone entities. Every inseperable Element inside a Block is connected to the block name with a double underscore ( __ ). Modifiers are used to change appearance, state or behaviour of a element. These Modifiers are connected with a double dash(--). Additional Modifiers are added with a single dash(-).

Head over to getbem.com for a more detailed convention


                .block__element--modifier-modifier
                .paragraph--strong
                .navigation__logo
                .footer__contactInformation--small
                .video-player__TitleName

SCSS

We use the the CSS Preprocessor "Sass".

Head over to sass-lang for more Information

It can be used in various ways, main Target is to make our CSS files more Readable, especially while using BEM. 


                .block{
                    &__head{
                        &__playbutton{
                            color:red;
                        }
                    }
                }

Units

  • Font size is set with pixels or rem.
  • Height of an object is set with % or vh.
  • Width of an object is set with % or vw.

Folder Structure

Webpack is used to coordinate and and compile all of our assets. All HTML pages are stored in the HTML folder

Reusable HTML files (head.html,footer.html,...) are stored in /src/partials.

Images, SCSS and JS Files belong in their own folder in /src.
